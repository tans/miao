# 创意喵平台 V1 已知问题

**版本**: V1.0  
**更新时间**: 2026-04-09

## 功能限制

### 1. 任务审核机制

**问题**: 商家创建任务后需要管理员审核才能上架

**影响**: 
- 商家创建任务后状态为"待审核"（status=1）
- 创作者无法看到待审核的任务
- 需要管理员账号审核或手动更新数据库

**临时解决方案**:
```bash
# 方案一：直接更新数据库（仅用于测试）
sqlite3 data/miao.db "UPDATE tasks SET status = 2, review_at = datetime('now'), publish_at = datetime('now') WHERE id = <task_id>;"

# 方案二：使用管理员账号审核（需要管理员权限）
curl -X PUT http://localhost:8888/api/v1/admin/task/<task_id>/review \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"action": "approve"}'
```

**计划改进**: V1.1 添加测试模式，自动审核通过

### 2. 文件上传功能

**问题**: 暂不支持实际文件上传

**影响**:
- 创作者提交作品时只能提供 URL 链接
- 无法直接上传视频文件

**当前方案**: 使用第三方存储（如七牛云、阿里云 OSS）上传后提供链接

**计划改进**: V2 添加文件上传功能

### 3. 支付系统

**问题**: 充值为模拟操作，不对接真实支付

**影响**:
- 商家充值不需要实际支付
- 创作者提现不会真实到账
- 仅用于演示和测试

**当前方案**: 
```bash
# 商家可以任意充值
POST /api/v1/business/recharge
{"amount": 10000}
```

**计划改进**: V2 对接支付宝/微信支付

### 4. 消息通知

**问题**: 无实时推送功能

**影响**:
- 用户需要主动刷新查看消息
- 无浏览器通知
- 无邮件/短信通知

**当前方案**: 
- 使用 `/api/v1/messages` 查看消息
- 使用 `/api/v1/messages/unread-count` 查看未读数

**计划改进**: V2 添加 WebSocket 实时推送

### 5. 搜索和过滤

**问题**: 搜索功能有限

**影响**:
- 任务大厅只支持基础关键词搜索
- 无法按价格、时间等排序
- 无高级筛选条件

**当前方案**: 使用 `keyword` 参数进行简单搜索

**计划改进**: V2 添加高级搜索和过滤

## 技术限制

### 6. 并发性能

**问题**: 未进行大规模并发测试

**影响**:
- 不确定高并发下的性能表现
- 可能存在数据库锁竞争

**建议**: 
- V1 适用于小规模测试和演示
- 生产环境建议进行压力测试

**计划改进**: V1.1 添加性能测试和优化

### 7. 数据库备份

**问题**: 无自动备份机制

**影响**:
- 数据丢失风险
- 无法回滚到历史状态

**临时解决方案**:
```bash
# 手动备份
cp data/miao.db data/miao_backup_$(date +%Y%m%d).db

# 定时备份（crontab）
0 2 * * * cp /path/to/data/miao.db /path/to/backup/miao_$(date +\%Y\%m\%d).db
```

**计划改进**: V1.1 添加自动备份功能

### 8. 日志系统

**问题**: 日志功能简单

**影响**:
- 难以追踪问题
- 无结构化日志
- 无日志轮转

**当前方案**: 使用 Gin 默认日志

**计划改进**: V2 使用 zap 或 logrus

## 安全问题

### 9. 密码强度

**问题**: 无密码强度验证

**影响**:
- 用户可以设置弱密码
- 存在安全风险

**当前限制**: 最少 6 个字符

**建议**: 使用强密码（包含大小写字母、数字、特殊字符）

**计划改进**: V1.1 添加密码强度检查

### 10. 频率限制

**问题**: 无 API 频率限制

**影响**:
- 可能被恶意刷接口
- 无防护机制

**临时方案**: 使用 Nginx 限流

**计划改进**: V1.1 添加 rate limiting

### 11. HTTPS

**问题**: 默认使用 HTTP

**影响**:
- 数据传输不加密
- 不适合生产环境

**解决方案**: 使用 Nginx 反向代理配置 HTTPS

**计划改进**: 提供 HTTPS 配置示例

## 用户体验

### 12. 错误提示

**问题**: 部分错误提示不够友好

**影响**:
- 用户可能不理解错误原因
- 技术细节暴露给用户

**示例**:
```json
{
  "code": 40001,
  "message": "参数错误: Key: 'TaskCreate.Title' Error:Field validation for 'Title' failed on the 'required' tag"
}
```

**计划改进**: V1.1 优化错误提示文案

### 13. 前端验证

**问题**: 前端表单验证不完整

**影响**:
- 用户可能提交无效数据
- 增加后端负担

**当前方案**: 依赖后端验证

**计划改进**: V1.1 完善前端验证

### 14. 移动端适配

**问题**: 移动端体验一般

**影响**:
- 部分页面在手机上显示不佳
- 操作不够便捷

**当前方案**: 使用 Bootstrap 5 响应式布局

**计划改进**: V2 优化移动端体验

## 业务逻辑

### 15. 任务取消

**问题**: 任务取消逻辑简单

**影响**:
- 已有认领的任务取消可能引起纠纷
- 退款逻辑不完善

**当前方案**: 
- 商家可以取消任务
- 冻结金额自动退回

**计划改进**: V2 完善取消和退款流程

### 16. 申诉系统

**问题**: 申诉功能基础

**影响**:
- 纠纷处理不够完善
- 无仲裁机制

**当前方案**: 基础的申诉创建和查看

**计划改进**: V2 完善申诉处理流程

### 17. 信用系统

**问题**: 信用评分机制简单

**影响**:
- 无法有效识别优质创作者
- 评分规则不够细致

**当前方案**:

- 4 个等级（青铜、白银、黄金、钻石）

**计划改进**: V2 完善信用评分算法

## 数据一致性

### 18. 事务处理

**问题**: 部分操作未使用事务

**影响**:
- 可能出现数据不一致
- 并发场景下有风险

**示例**: 任务认领、审核等涉及多表操作

**计划改进**: V1.1 完善事务处理

### 19. 乐观锁

**问题**: 无乐观锁机制

**影响**:
- 并发更新可能覆盖数据
- 余额扣减可能出错

**计划改进**: V1.1 添加版本号字段

## 总结

V1 版本是一个 MVP（最小可行产品），专注于核心功能的实现。上述问题大部分不影响基本使用，但在生产环境使用前需要进行相应的改进和完善。

### 优先级分类

**高优先级（V1.1）**:
- 任务审核机制优化
- 密码强度验证
- API 频率限制
- 事务处理完善
- 错误提示优化

**中优先级（V2）**:
- 文件上传功能
- 支付系统对接
- 实时消息推送
- 高级搜索过滤
- 性能监控

**低优先级（后续版本）**:
- 移动端优化
- 完善申诉系统
- 信用系统升级
