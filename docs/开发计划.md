# 创意喵 2.0 开发计划

## 一、版本规划

### V1.0 MVP（当前阶段）
**目标**：最小可用产品，验证核心撮合流程

**时间估算**：约 4-6 周

**核心功能**：
1. 用户系统（商家、创作者、管理员三级）
2. 任务发布与管理（商家端）
3. 任务浏览与投稿（创作者端）
4. 基础资金托管
5. 简单的审核流程
6. 管理员后台基础功能

---

### V2.0 完善阶段
- 双向信用体系
- AI 风控基础（内容查重）
- 申诉流程
- 完整的资金结算
- 数据统计

---

### V3.0 扩展阶段
- 消息通知系统
- 高级数据分析
- 运营工具完善
- 移动端适配

## 二、技术架构

### 技术栈
| 层级 | 技术选型 |
|------|---------|
| 后端 | Go + Gin |
| 数据库 | SQLite |
| 前端 | HTML + Bootstrap 5 |
| 认证 | JWT |
| API | RESTful /api |

### 项目结构
```
miao/
├── cmd/
│   └── server/
│       └── main.go
├── internal/
│   ├── config/
│   ├── handler/        # HTTP handlers
│   ├── middleware/     # 中间件（认证、日志、跨域）
│   ├── model/         # 数据模型
│   ├── repository/    # 数据访问层
│   ├── service/       # 业务逻辑层
│   └── router/        # 路由配置
├── pkg/
│   └── utils/         # 工具函数
├── web/
│   ├── static/        # CSS, JS, 图片
│   └── templates/     # HTML 模板
├── migrations/        # 数据库迁移
├── docs/             # 文档
├── go.mod
└── go.sum
```

## 五、API 设计规范

### 基础格式
```
/api/v1/{模块}/{操作}
```

### 认证方式
- Header: `Authorization: Bearer <token>`

### 响应格式
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 核心 API 列表

#### 认证模块 `/api/v1/auth`
- [ ] `POST /register` - 用户注册
- [ ] `POST /login` - 用户登录
- [ ] `POST /logout` - 退出登录

#### 用户模块 `/api/v1/users`
- [ ] `GET /profile` - 获取个人信息
- [ ] `PUT /profile` - 更新个人信息
- [ ] `GET /credits` - 获取信用分

#### 任务模块 `/api/v1/tasks`
- [ ] `GET /` - 任务列表（创作者）
- [ ] `POST /` - 创建任务（商家）
- [ ] `GET /:id` - 任务详情
- [ ] `PUT /:id` - 编辑任务
- [ ] `DELETE /:id` - 终止任务
- [ ] `GET /my` - 我的任务（商家）

#### 投稿模块 `/api/v1/submissions`
- [ ] `POST /` - 提交投稿
- [ ] `GET /` - 投稿列表
- [ ] `PUT /:id/review` - 审核投稿
- [ ] `PUT /:id/award` - 评选获奖

#### 资金模块 `/api/v1/account`
- [ ] `GET /balance` - 余额查询
- [ ] `POST /recharge` - 充值
- [ ] `GET /transactions` - 流水记录

#### 申诉模块 `/api/v1/appeals`
- [ ] `POST /` - 发起申诉
- [ ] `GET /` - 申诉列表
- [ ] `GET /:id` - 申诉详情

#### 管理模块 `/api/v1/admin`
- [ ] `GET /dashboard` - 数据看板
- [ ] `GET /users` - 用户列表
- [ ] `PUT /users/:id/credit` - 修改信用分
- [ ] `PUT /users/:id/status` - 账号状态

## 八、注意事项

1. **安全**：密码加密存储、JWT 有效期控制、SQL 注入防护
2. **性能**：SQLite 并发限制、合理的索引
3. **体验**：前端表单验证、友好的错误提示
4. **可维护**：代码分层清晰、注释完善
