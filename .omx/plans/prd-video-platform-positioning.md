# PRD: 创意喵单一视频平台收敛

## 背景

创意喵当前的公开定位已经偏向“视频创意任务平台”，但实际产品仍以多分类创意市场运行，导致用户预期、页面表达、后端逻辑与测试口径不一致。

## 问题陈述

- 用户在说明文档里看到的是视频平台。
- 用户在实际操作里看到的是文案、设计、摄影、音乐、开发等多分类任务市场。
- 后端仍接受任意分类写入，测试也持续为多分类背书。

## 目标

把产品彻底收敛为单一视频任务平台，让用户可见层、文档层、后端默认行为、测试口径全部一致。

## 非目标

- 不做历史任务数据迁移。
- 不新增复杂的视频结构化字段。
- 不重构无关业务流程。

## 用户故事

### US-001 商家发视频任务

作为商家，我希望发布任务时直接面向视频创作，而不是先从多个创意分类里做选择，这样我能更明确地描述视频需求。

验收标准：
- 发单页不再出现任务分类下拉框。
- 发单页文案围绕视频脚本、镜头、剪辑、交付格式展开。
- 后端保存任务时统一按视频分类落库。

### US-002 创作者浏览视频任务

作为创作者，我希望任务大厅只呈现视频任务，不再被无关分类干扰，这样我能更快筛选合适的视频订单。

验收标准：
- 创作者任务大厅不再提供分类筛选。
- 列表、筛选标签、统计信息、详情页不再展示多分类标签。
- 价格、排序、关键词搜索继续可用。

### US-003 访客理解平台定位

作为访客，我希望从 README、公开任务大厅和产品文案中清楚知道这是视频任务平台，而不是泛创意众包市场。

验收标准：
- README 和 PRD 不再宣称平台支持多类创意任务。
- 公共任务大厅不再暴露分类筛选。
- 页面说明统一使用视频任务口径。

## 范围

### In Scope

- 文档统一视频定位
- 商家发单页去分类化
- 创作者任务大厅去分类化
- 公共任务大厅去分类化
- 任务详情与任务列表去分类标签
- 后端创建任务强制归一到视频分类
- 测试脚本从多分类筛选转为视频任务流验证

### Out of Scope

- 历史数据修复
- 新的视频元数据模型
- UI 大改版

## 约束

- 不新增依赖。
- 不覆盖现有未提交但与本任务无关的变更。
- 保持接口与存储层尽量兼容，优先选择低风险收敛方案。

## 成功指标

- 用户可见界面不再出现多分类入口或多分类标签。
- 新建任务统一按视频分类写入。
- 文档与产品表达一致。
- 相关验证通过，无明显回归。
