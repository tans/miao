# Test Spec: 创意喵单一视频平台收敛

## 测试目标

验证产品从“多分类创意市场”收敛为“单一视频任务平台”后，文档、后端默认行为、页面表现和回归风险都处于可接受状态。

## 测试范围

### 1. 后端默认行为

- 创建任务时，即使请求传入非视频分类，保存结果也会归一为视频分类。
- 任务模型的对外常量和说明不再暗示平台支持多分类业务。

### 2. 页面表现

- 商家发单页没有任务分类选择控件。
- 创作者任务大厅没有分类筛选控件。
- 公共任务大厅没有分类筛选控件。
- 创作者详情页和商家任务列表不再展示多分类标签。

### 3. 文档与文案

- README、PRD 文档不再宣称支持文案/设计/摄影/音乐/开发等多分类市场。
- 页面帮助文案围绕视频创作展开。

### 4. 回归项

- 任务大厅的价格筛选、排序、关键词搜索仍保留。
- 测试脚本语义改为视频任务流后仍可通过基础语法校验。

## 测试方法

### 单元测试

- `go test ./internal/model -run TestNormalizeTaskCategoryToVideoOnly -v`

目的：
- 验证任意输入分类都会被归一为视频分类，防止后续回归。

### 静态模板回归检查

- `rg -n "任务分类|文案创作|设计创意|摄影拍照|音乐音频|程序开发" web/templates/business/task_create.html web/templates/creator/task_hall.html web/templates/tasks.html web/templates/creator/task_detail.html web/templates/business/task_list.html`

期望：
- 不再命中用户可见的多分类入口或标签。

### 文档回归检查

- `rg -n "文案创作|设计创意|摄影拍照|音乐音频|程序开发|多分类创意" README.md docs/V1_0_PRD.md`

期望：
- 不再存在与单一视频平台定位冲突的表达。

### 脚本校验

- `bash -n test/test_task_filters.sh`

期望：
- 脚本语法有效。

### 全量 Go 回归

- `go test ./...`

期望：
- 受本次改动影响的 Go 代码无编译或测试回归。

## 通过标准

- 单元测试通过。
- 模板与文档扫描符合预期。
- 脚本语法校验通过。
- Go 测试通过，或明确记录现存非本次引入的问题。
