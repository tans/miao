# miao 后端人工测试案例流程文档

> 基线版本：当前仓库代码  
> 编写日期：2026-04-27  
> 适用对象：研发、测试、联调

## 1. 文档目标

本文档用于指导 `miao` 后端接口的人工回归测试，重点覆盖与小程序和管理后台共用的业务链路：

- 用户登录、权限鉴权、管理员权限
- 任务、认领、作品、灵感、申诉的增删改查
- 财务、充值、提现、交易流水
- 数据库表管理与 SQL 调试

## 2. 测试环境与前置条件

### 2.1 环境

- 项目目录：`/data/miao`
- 本地启动：`go run ./cmd/server`
- 默认接口前缀：`/api/v1`
- 管理员接口前缀：`/api/v1/admin`

### 2.2 必要配置

- `JWT_SECRET` 必须已配置
- 本地数据库可正常连接，默认使用 `./data/miao.db`
- 管理员账号可正常登录

### 2.3 测试数据建议

- 至少 3 个用户：商家、创作者、管理员
- 至少 3 个任务：待审核、进行中、已结束
- 至少 3 条认领：已认领、已提交、已审核
- 至少 1 条作品、1 条申诉、1 条交易流水
- 至少 1 个可查询的数据库表

## 4. 通用测试流程

### TC-GEN-001 登录成功

- 前置条件：管理员账号可用
- 步骤：
  1. 调用 `POST /api/v1/admin/login`
  2. 传入正确的用户名和密码
- 预期结果：
  1. 返回 `code=0`
  2. 返回管理员 token
  3. 后续 admin 接口可正常访问

### TC-GEN-002 未登录拦截

- 前置条件：不携带 token
- 步骤：
  1. 访问任一 `admin` 接口
- 预期结果：
  1. 返回 401 或 403
  2. 不返回业务数据

### TC-GEN-003 非管理员拦截

- 前置条件：使用普通用户 token
- 步骤：
  1. 调用任一 `admin` 接口
- 预期结果：
  1. 返回无权限
  2. 不允许越权读取管理数据

## 5. 数据概览

### TC-DASH-001 指标加载

- 前置条件：数据库中存在真实业务数据
- 步骤：
  1. 调用 `GET /api/v1/admin/dashboard`
  2. 调用 `GET /api/v1/admin/stats`
- 预期结果：
  1. 返回总用户、总任务、进行中任务、总作品、待审核任务、待处理申诉、收入统计
  2. 指标值与数据库一致

## 6. 用户管理

### TC-USER-001 用户列表

- 步骤：
  1. 调用 `GET /api/v1/admin/users`
  2. 按角色、状态、关键词筛选
- 预期结果：
  1. 列表返回正常
  2. 筛选条件生效

### TC-USER-002 用户详情

- 步骤：
  1. 调用 `GET /api/v1/admin/users/:id`
- 预期结果：
  1. 返回基础资料、余额、认证信息、统计信息
  2. 不存在用户时返回明确错误

### TC-USER-003 状态与余额调整

- 步骤：
  1. 调用 `PUT /api/v1/admin/users/:id/status`
  2. 调用 `PUT /api/v1/admin/users/:id/balance`
  3. 调用 `PUT /api/v1/admin/users/:id/credit`
- 预期结果：
  1. 状态、余额、信用分更新成功
  2. 余额变更生成交易流水

## 7. 任务与认领
