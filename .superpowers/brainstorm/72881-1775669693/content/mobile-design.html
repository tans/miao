3. 组件与交互

  • 统一使用移动端容器，宽度围绕手机视口设计，不再沿用 PC 左侧边栏。
  • 任务卡改成封面图/色块 + 标题 + 赏金 + 截止时间 + 操作按钮。
  • 筛选降级为顶部轻量 chips，不做复杂后台筛选面板。
  • 我的页用宫格入口 + 列表入口，兼顾常用操作与账户信息。
  • 保留现有 API 能力，优先改模板层与前端组织方式。

4. 技术落点

建议新增独立的 /mobile/ 模板目录和静态资源入口，和现有桌面端并存，不做原页面直接覆盖。首批优先落地：

  • /mobile/index.html：任务大厅
  • /mobile/showcase.html：过审作品
  • /mobile/me.html：我的
  • 从「我的」进入发布任务、领取任务、钱包等二级页

5. 风险与范围

先做的

  • 独立移动端入口
  • 3-tab 主框架
  • 核心页面 app 化改造
  • 现有 API 复用

暂不做的

  • 完全重写后端模型
  • 复杂双端状态同步策略
  • 原有桌面端统一重构
